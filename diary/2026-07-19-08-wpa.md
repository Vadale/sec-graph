# 2026-07-19 — Session 08 — WP-A (SQLAlchemy rules pack + TRR)

## Goal
Model SQLAlchemy so ORM query-builder calls become legible (rule-classified, taint flows
through the chain), and instrument the TRR (taint-relevant resolution) metric.

## Done
- `rules/python/sqlalchemy.yml`: query-builder methods (`.filter/.where/.all/.scalars/...`) as
  **propagators**, `sqlalchemy.text` as a propagator, `.execute/.exec_driver_sql` as the raw-SQL
  sink. ORM `.filter(...)` is parameterized (safe) so it is a propagator, not a sink.
- Engine: **propagators now carry the receiver's taint** (a sound over-approx) so a query-builder
  chain (`q.filter(t).all()`) keeps taint through the terminal call.
- **TRR instrumentation:** `run_function_inter` returns the tainted call sites; `run_project_full`
  aggregates them; `secgraph.callgraph.trr` computes TRR; `callgraph-stats` prints PCR / UNK / TRR.
