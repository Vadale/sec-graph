# 2026-07-19 — Session 11 — WP-C2 (auth barriers + unguarded sinks)

## Goal
The other half of the Phase-4 layer-tagger: detect auth barriers and surface the flagship derived
finding — the **unguarded sink** (a dangerous sink with no auth barrier on the path). Governing
rule: a false "guarded" hides an unguarded sink (a security FN), so never credit an unproven guard.

## Done (ADR-010)
- **`secgraph/taint/guards.py`** (new): `guard_map(fn, imap, rules)` -> per-statement auth guards.
  Three detectors — B1 decorators, B2 authorised-arm, B3 terminating gate — with B2/B3 unified by a
  **polarity analysis** (`_true_guards`/`_false_guards` thread `and`/`or`/`not` by De Morgan).
- **Engine**: `SinkPoint.guards`/`Finding.guards`; `_lift` unions call-site guards across a hop;
  `_emit` + `interproc` merge on key collision by **intersection** (guarded only if every path is);
  `find_unguarded_sinks`. `guard_map` is purely structural -> summary monotonicity + determinism hold.
- **IR/rules**: `FunctionIR.decorators` (harvested from the `decorated_definition` parent);
  `BarrierRule` (decorators/callables/test_attrs/aborts) + `rules/labels.yml` barriers.
- **Surfacing**: `guards`+`unguarded` in taint.json; viz UNGUARDED badge + guard chip; CLI unguarded
  count (`analyze` prints "N finding(s), M unguarded").

## Reviewer → fixes (two ship-blocking security FNs, both fixed + regression-tested)
- **FINDING 1 (CRITICAL)**: `if authed or debug:` (and `if flag and not authed:`, auth term in a
  comparison / as a call arg) was falsely "guarded" because detection took the first auth term and
  only checked a top-level `not`. Fixed by the polarity-sound `_true_guards`/`_false_guards`.
- **FINDING 2 (HIGH)**: in a recursive SCC, `interproc`'s keep-first `setdefault` froze an early
  "guarded" verdict over a later unguarded path. Fixed by intersection-merge across iterations.
- **FINDING 3 (MEDIUM)**: `HttpResponseForbidden` (a constructor) removed from `aborts` — it does not
  terminate unless returned (and `return HttpResponseForbidden(...)` is already caught by `Return`).
- FINDING 4 (cosmetic determinism nit on merged non-guard fields): noted, deferred; guards stay
  deterministic. Reviewer confirmed monotonicity, cross-hop-no-leak, decorator harvest all sound.
