# 2026-07-19 — Session 09 — WP-B (Phase 4: projection + layered viz)

## Goal
Turn the interprocedural findings into the two artifacts + the demo: annotate graphify's
`graph.json` (coarse, sidecar-safe), emit `taint.json` (fine), and render the self-contained
layered HTML map. Wire `secgraph analyze` to produce all three.

## Done
- **`secgraph/project.py`** — `analyze_project(path, out_dir)`: run interprocedural taint, run
  graphify through the adapter, annotate `graph.json` in place, write `taint.json`, render
  `secgraph.html`.
  - `_annotate_graph_json`: adds `sec_layers` to the source/sink **function** nodes and one
    `sec-path-N` hyperedge (`kind: "taint-path"`) per finding. **Node count left unchanged**
    (sidecar discipline) — asserted.
  - Findings are mapped to graphify nodes **structurally**, never by name: reuse `ir.join`
    (`join_modules`, by `(source_file, def-line)`) to bind `FunctionIR.graphify_node`, then map
    each finding's statement line to its enclosing function by **tightest span containment**
    (`_enclosing_node`).
- **`secgraph/viz.py`** — `render_html`: single self-contained HTML (inline CSS/JS, no external
  resources), a layer checkbox per security layer (Google-Maps toggles), one card per finding
  (source→sink railway + real code slices + trace). XSS-safe (`textContent`) **and** script-data
  safe (findings embedded as JSON with `<` → `<`).
- **`secgraph/cli.py`** — `analyze` → `analyze_project` (prints the 3 files + finding count);
  `view` opens `secgraph.html`.
- Fixtures/tests: `tests/fixtures/methods/{view,dao}.py` (method endpoint → cross-file SQLi);
  `tests/project/test_projection.py` (6 tests).

## Reviewer → fixes (all verified)
The `reviewer` agent found the join was by **function name**, which silently annotated **nothing
for methods** (graphify labels a method `.get()`, our IR names it `get`) and overwrote on
same-name collisions — the graphify/MCP-facing half of the deliverable was empty for most real
Python. Fixed by the structural span-join above; regression test
`test_method_hosted_finding_annotates_the_method_node` proves the `.get()` node is now annotated
and joined by a cross-file hyperedge. Also fixed: HTML script-data breakout on a slice containing
`<!--<script>` (MEDIUM → `<`-escape, regression test) and the `file_type=="code"` filter gap
(LOW → structural, `join_modules` already filters). `simplify-pass`: no changes (already minimal),
65 == 65.
