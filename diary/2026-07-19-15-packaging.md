# 2026-07-19 — Session 15 — C: Phase 8 packaging

## Goal
Make sec-graph a `pip install`-able wheel that ships its runtime data — the YAML rule packs and the
viz assets — and resolves them whether run from an editable dev checkout or an installed release.

## Done (ADR-013)
- `pyproject.toml`: `[tool.hatch.build.targets.wheel] artifacts` ships `secgraph/viz/*.css|*.js`;
  `force-include` maps the repo-root `rules/` into the wheel at `secgraph/_rule_packs`. Version
  `0.0.0 → 0.1.0` (alpha).
- `rules/loader.py::default_rules_dir()` dual-resolves: the repo-root `rules/` in a dev checkout,
  else `importlib.resources.files("secgraph")/"_rule_packs"`. The layout (`rules/` at root) is
  unchanged — no churn to CLAUDE.md/docs/tests.

## Acceptance (executed)
```
$ uv build --wheel  →  secgraph-0.1.0-py3-none-any.whl
  contains: secgraph/_rule_packs/{labels,secrets}.yml + python/{flask,sqlalchemy}.yml,
            secgraph/viz/map.css, secgraph/viz/map.js
$ uv venv fresh; uv pip install <wheel>; cd /outside/the/repo
$ secgraph analyze <fixture>  →  graph.json + taint.json + secgraph.html (37 KB) + 1 finding
  default_rules_dir() -> …/site-packages/secgraph/_rule_packs  (the PACKAGED rules, all 4 yml)
$ pytest -q  (dev)  →  104 passed;  dev default_rules_dir() still -> repo-root rules/
```
So the wheel is self-standing: installed anywhere, `secgraph analyze` works with the bundled rules +
viz, no dev tree needed.
