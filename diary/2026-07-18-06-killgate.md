# 2026-07-18 — Session 06 — KILL-GATE resolution (thesis validated)

## What happened
Ran the Phase-3 KILL-GATE benchmark on 3 real repos (microblog, fastapi-realworld,
flask-realworld). Raw binding rate: **10-17% — apparent fail.** Diagnosed it, consulted a
**Fable 5 max** agent, re-specified the gate (ADR-007), fixed the classifier, and
re-measured: **PCR 94.7-100% — gate PASSES. Build-on-graphify thesis validated.**

## Diagnosis
The raw 10-17% was **classifier failure, not resolution failure**: 83% of "unresolved" were
SQLAlchemy/library method calls (`db.session.scalar`, `query.where`) mis-labelled "unresolved"
because the receiver (`db`) traced through a project package; 17% were project class
constructors we didn't index. Project *function* calls always resolved fine.

## Fix (ADR-007 + classifier v2)
- **ADR-007** supersedes the single-rate gate with three metrics over a strict partition
  (`rule|builtin|bound|external|unknown-receiver|unresolved-project`): **PCR** (project-call
  resolution) >=85%, **UNK** (unknown-receiver / method sites) <=40% reported, **TRR** (taint-
  relevant) >=70%. `external` requires positive evidence; no-evidence -> `unknown-receiver`
  (anti-gaming).
- **Value-origin tracking** (`ModuleIR.globals`): module-level `db = SQLAlchemy()` lets the
  resolver tell a library object (`db.session.x` -> external) from a project singleton.
- **Class index + constructors** (`ModuleIR.classes`, `FnIndex.classes`): `User()` binds;
  `self.method` / `v = User(); v.method()` resolve via a Tier-1 local-type pass.
- **Transitive external bases**: `Article -> project Model -> db.Model` correctly -> external.
- The taint ENGINE is untouched: only project-function binds feed summaries, so **50 tests
  stay byte-identical** (constructor/method binds are counted for the metric; their taint
  propagation is WP3-b).

## Result (ADR-007 metric, PCR gate = 85%)
```
                    PCR      unresolved-project   UNK
microblog          100.0%    0                    11.1%
fastapi-realworld   99.4%    1                    17.7%
flask-realworld     94.7%    4                    17.6%
```
Honesty-checked: `unresolved-project` (genuine misses) is 0/1/4; `unknown-receiver` (11-18%)
is correctly the untyped SQLAlchemy query-builder chains (`q.where/all/order_by` on locals we
can't type) — not hidden project failures.

## Verdict
The gate passes decisively; the raw number was an artifact. **No graphify divorce.** The
remaining blind mass is library query-builder methods -> a rules-pack (YAML) problem, exactly
as ADR-007's kill-criterion says.

## Deferred (WP3-b)
Wire the resolved method/constructor binds into the taint ENGINE (constructor-taint rule:
result = fallback-over-approx UNION summary, never trust empty return_params; the arg->param /
receiver-param map; `Call.kw_names`); TRR instrumentation; more type tiers; SQLAlchemy/WTForms
rules packs; the graphify `calls` differential validator; a pinned `tests/benchmarks/` harness.
