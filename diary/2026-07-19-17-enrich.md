# 2026-07-19 — Session 17 — Phase 10 (layer enrichment over ingested findings)

## Goal
The pivot's differentiator (ADR-014): ingested SARIF/semgrep findings gain what no SAST emits —
credentials/PII layers, secret classification, and the auth/unguarded verdict — honestly labelled.

## Done
- **`secgraph/ingest/enrich.py`** (graphify-free): `enrich_findings(findings, root, modules, rules)`:
  - **lexical sensitive-data layers**: identifiers on the finding's source/sink/hop lines →
    `ident_label`; quoted literals → `classify_secret`. Add-only to `layers`, `enrich:lexical@…`
    provenance. Identifier-on-a-verified-flow, not taint-on-the-value — labelled as such.
  - **auth/unguarded**: for a Python sink, `guard_map` (ADR-010) over the enclosing `FunctionIR`
    (from the project IR already built in `analyze_ingest`), guards at the sink statement's sid
    (`_sid_at_line`) → `guards`/`unguarded`/`guard_status:"analyzed"`.
  - **honest tri-state**: everything else (non-Python sink, no enclosing fn) stays
    `guard_status:"unknown"` (`unguarded:false, guards:[]`).
- **Consumers render the tri-state without a false claim**: `map.js markGuard` returns early on
  `unknown` (no glow, **no** green ring; badge "guards ?"); `mcp_view` orders unguarded < unknown <
  guarded, `find_unguarded_sinks` excludes unknowns by default (+`unknown_count`, `include_unknown`);
  `_summary` carries `guard_status`. The built-in engine path is bit-identical (no `guard_status` key
  → treated as "analyzed").
- Wired into `analyze_ingest` (after ingest, before projection).
