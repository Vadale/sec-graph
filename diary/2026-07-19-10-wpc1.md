# 2026-07-19 — Session 10 — WP-C1 (sensitive-data layers + f-string fix)

## Goal
Add the deterministic **credentials/PII** layers so the flagship query works —
`credentials ∈ path.layers AND dangerous-sink ∈ path.layers` ("secrets into dangerous sinks") —
and fix the f-string taint false negative found along the way.

## Design (Fable 5 max consulted, ADR-009)
Layers ride the **value**, not the IR node: sensitive-data evidence **mints a fresh label-`Origin`**
into the taint state (`Finding.layers = origin.layers ∪ sink.layers` already existed) and **never
mutates** an existing origin — the invariant that keeps summary monotonicity and byte-determinism.
Four localized mint sites in `engine.py`; label flow across functions is then free via summaries.

## Done
- **`secgraph/rules/labels.py`** (new): `ident_label` (word-based token-run match + plural),
  `classify_secret` (named formats → entropy fallback), `luhn`. Pure; returns `(layers, conf)` so
  the engine builds the Origin (rules stay independent of taint).
- **Engine mint sites** (`engine.py`): source subscript key (`request.form['password']`),
  credential/PII-named assignment/for/walrus target, credential/PII-named parameter (unconditional),
  secret literal + module constant. `TaintCtx.constants`, threaded from `interproc.py`.
- **f-string fix** (`lower.py`): a string with interpolations lowers to `Unknown("fstring", …)`
  (was an opaque `Literal`), plus `_string_content` and `_module_constants`. IR carriers
  `Literal.text`, `ModuleIR.constants`.
- **Rules**: `LabelRule`/`SecretPattern`/`SecretConfig` (model + loader); `rules/labels.yml`
  (credentials/pii identifier dicts) + `rules/secrets.yml` (named regexes + entropy config).

## Reviewer → fixes (all verified)
Reviewer confirmed the **risky invariants are sound by construction** (mint-don't-mutate,
determinism, monotonicity, f-string over 9 shapes, Luhn). Fixed 4 precision/recall items (Accuracy
is pillar #1): (#1) module constant leaking through a shadowing local → `constants` shadow-filter;
(#2) hex entropy fallback flagging hashes/decimal-ids → exclude md5/sha lengths + require an a–f
char + threshold 3.0→3.5; (#3) deny-value over-suppression → match deny against the pattern's
matched span, not the whole literal; (#4) plurals (`passwords`, `api_keys`) missed → plural-tolerant
token match. Each has a regression test.
